## [1. 주요 기능 소개]

이 서비스의 핵심 기능은 크게 네 가지입니다.

첫째, **학번/사번 기반 로그인**입니다. 최초 로그인 시 비밀번호를 변경하는 First Setup 절차가 포함되어 있습니다.

둘째, **수강신청 시스템**입니다. 강의 조회, 장바구니 담기, 수강신청 및 취소가 가능하고, 정원 초과 시 대기열 처리도 구현했습니다.

셋째, **관리자 기능**입니다. 과목 생성과 삭제, PDF 강의계획서 업로드, 수강신청 일정 관리를 관리자 페이지에서 처리할 수 있습니다.

넷째, **RAG 기반 AI 챗봇**입니다. 학사 규정 문서를 벡터 검색하여 학생 질문에 답변하고, AI 과목 추천도 제공합니다.

## [2. 기술 스택]

백엔드는 Python 3.11과 FastAPI를 사용했고, SQLAlchemy ORM으로 데이터베이스를 관리합니다.

프론트엔드는 별도 프레임워크 없이 Vanilla HTML/CSS/JavaScript로 구성했으며, Nginx가 정적 파일을 서빙합니다.

인증은 JWT와 bcrypt를 사용합니다.

데이터베이스는 PostgreSQL을 메인으로 사용하고, pgvector 확장을 통해 RAG 벡터 검색도 처리합니다.

AI 기능은 AWS Bedrock을 통해 임베딩 생성과 LLM 답변을 처리합니다.

인프라는 Docker와 Terraform으로 구성했으며, GitHub Actions로 CI/CD 파이프라인을 자동화했습니다.

## [3. AWS 인프라 아키텍처]

AWS 구성을 설명하겠습니다.

서울 리전(`ap-northeast-2`)에 VPC를 구성하고, 퍼블릭 서브넷에 Nginx 프록시 EC2를 두었습니다.
실제 앱 서버는 프라이빗 서브넷의 Blue/Green ASG EC2에서 실행됩니다.

배포 시 프록시의 Nginx 업스트림을 Blue에서 Green으로 전환하는 방식으로 무중단에 가깝게 운영합니다.

데이터베이스는 RDS PostgreSQL을 사용합니다. 프라이빗 구조라 외부에서 직접 접근할 수 없고, 운영 점검 시에는 SSM 포트포워딩으로 접속합니다.

컨테이너 이미지는 ECR에 저장되고, Terraform 상태는 S3 백엔드로 관리합니다.

## [4. DynamoDB에 대해]

아키텍처 다이어그램에서 DynamoDB가 표시되어 있는데, 현재는 실제로 연결해 사용하고 있지 않습니다.

챗봇 대화 이력을 DynamoDB에 저장하도록 Terraform 인프라는 `mugang-chat-history` 테이블로 미리 설계해 두었습니다.
다만 현재는 소규모 운영이라 PostgreSQL로 충분하고, DynamoDB를 연결하면 추가 비용이 발생합니다.
그래서 실제 연결은 서비스 트래픽이 늘어나는 시점에 붙일 예정입니다.

현재 챗봇 이력은 PostgreSQL에 저장되고 있으며, 기능적으로는 완전히 동작합니다.

## [5. CI/CD 파이프라인]

GitHub Actions를 통해 `main` 브랜치에 push가 발생하면 자동으로 배포가 진행됩니다.

흐름은 다음과 같습니다.
- Frontend와 Backend Docker 이미지를 빌드
- ECR에 이미지 푸시
- Terraform으로 Blue/Green 색상 전환
- Proxy Nginx 업스트림을 신규 색상으로 스위칭

전체 리소스를 정리할 때는 별도 `destroy.yml` 워크플로를 수동으로 트리거합니다.

## [7. 향후 개선 계획]

네 가지 개선 방향을 잡고 있습니다.

- **DynamoDB 연결**: 챗봇 대화 이력 저장을 PostgreSQL에서 DynamoDB로 이전하여 비용 효율적인 NoSQL 구조로 전환
- **Alembic 마이그레이션**: 현재 `create_all` 방식에서 버전 관리 가능한 마이그레이션 도입
- **테스트 자동화**: pytest 기반 API 시나리오 테스트 구축
- **배포 표준화**: K8s 매니페스트와 Terraform 경로 중 하나로 통일
